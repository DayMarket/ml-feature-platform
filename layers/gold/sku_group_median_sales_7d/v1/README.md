# Gold Median Sales 7D по SKU Group ID

Пайплайн собирает фичу медианного дневного количества продаж за последние 7 суток на уровне `sku_group_id`.

Целевая таблица: `iceberg.gold.feature_platform_sku_group_median_sales_7d`.

Основная логика:

- читает заказы из `iceberg.silver.order_items`;
- обогащает заказы SKU-данными из `iceberg.silver.sku`;
- берет завершенные продажи за последние 7 суток до `data_interval_end`;
- исключает позиции, возвращенные до окончания расчетного интервала;
- раскладывает окно на семь 24-часовых бакетов;
- заполняет пропущенные бакеты нулевыми продажами для sku group, у которых были продажи в окне;
- считает медиану продаж по семи бакетам;
- пишет результат в Iceberg через `overwritePartitions()`.

DAG запускается раз в три часа. В отличие от дневных пайплайнов, локальная `config/factory.py` передает `data_interval_start` и `data_interval_end`, чтобы каждый трехчасовой запуск считал актуальный rolling-window срез.
